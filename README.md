## 2주차 타이쿤 회의록

### 📋 캡스톤디자인 프로젝트 시나리오 정하기<br>

#### 1. 회의개요

+ 일시: 3월 14일 12:00 ~ 1:00<br>
+ 장소: 조선대학교 IT(3120)강의실<br>
+ 참석자: 김윤호, 노수연, 이하연, 최예린<br>
+ 작성자: 최예린

#### 2. 회의 내용
1) 아이디어

   - 컴퓨터공학과의 일상을 담은 스토리(최예린)

#### 3. 결정사항<br>
<details>
  <summary><span style="font-weight: bold; font-size:15px; color:gray">🎬 게임 시나리오[클릭]</span></summary>
<div markdown="1">       

<span style="font-weight:bold; font-size:20px">게임 아이디어 정리</span>

1. 컨셉 : 컴퓨터공학과
2. 구성
   1) 맵
  
      - 시험장 : 보스가 나타나는 장소
      - 101 ~ 103 : 난이도별 몬스터 스폰 장소
      - 로비 : 커피(피로도) & 음식(체력)을 구매할 수 있는 장소
      - 연구실 : NPC(퀘스트)와 컴퓨터(공격력) 업그레이드가 가능한 장소
   2) 캐릭터
  
      - 레벨 : 1학년 ~ 4학년
      - 체력 : 체력이 없으면 죽음 + 쿨타임 & 로비에서 스폰
      - 피로도 : 피로도가 꽉차면 아무것도 안하고 로비로 돌아감
      - 공격력 : = (레벨 + 경험치) * 컴퓨터 성능
      - 경험치 : 프로그래밍 능력으로 다 채우면 다음 학년으로 진급
      - 골드 : 음식과 커피, 컴퓨터를 업그레이드 시킬 수 있음
   3) NPC
  
      - 교수님(정현숙) : 과제를 부여하며 보상으로 골드와 경험지를 줌 / 시험장 입장 및 진급 도우미
   4) 사물
  
      - 커피 자판기 : 다양한 종류의 커피를 파는 곳(원두에 따라 피로도 회복량이 달라짐)
      - 음식 가판대 : 다양한 음식을 파는곳(음식 종류에 따라 체력 회복량이 달라짐)
      - 컴퓨터 : 컴퓨터의 부품을 업그레이드 시킬 수 있음(공격력 증가)
   5) 몬스터
   
   |장소|난이도|1학년|2학년|3학년|4학년|
   |:--:|:--:|:--:|:--:|:--:|:--:|
   |101|하|Python|JAVA|컴퓨터구조|인공지능 트랙|
   |102|중|C|데이터구조|컴퓨터네트워크|정보보호 트랙|
   |103|상|C++|알고리즘|데이터베이스|IoT 트랙|
   |시험장|중간/기말|특수 몬스터|
  
      - 각 몬스터는 경험치와 골드 드랍
      - 일정 확률로 아이템(컴퓨터 업그레이드 부품) 드랍

   6) 이벤트
  
      - 중간고사 : 경험치가 50%가 되었을 때 발생
      - 기말고사 : 경험치가 100%가 되었을 때 발생
      - 기말고사를 끝내고, 다음 학년이 될 때는 일정 골드(학비)를 지급하여야만 진급됨



3. 경험치
  
        - 경험치는 각 캐릭터 개인이 가진 것이 아닌 통합 경험치로 이루어져 각 캐릭터가 사냥하며 얻은 경험치들이 합산되어 나타난다.
      - 해당 경험치의 중간중간(50%, 100%)에 보스 몬스터가 소환되며 해당 몬스터를 물리쳐야 진행할 수 있다.

4. 최대인원(각 학년이 될 때마다 최대인원이 풀리고, 골드로 인원을 늘릴 수 있다.)
  
        - 1학년 때    : 1명
      - 2학년 때    : 3명
      - 3학년 때   : 4명
      - 4학년 때   : 5명 


</div>
</details>

그래픽 작업 시작

    - 김윤호: 배경과 게임 캐릭터 사이의 물리적인 오류 해결방법 구상중(추후 알고리즘을 통해 기존의 방식과는 다른 방법으로 물리적 오류 해결 진행 계획, 몬스터는 구현 성공)
    - 이하연: 게임 시작 화면 구현 완료 및 창(샌드위치, 커피, 연구실 컴퓨터) GUI 구현 완료
    - 최예린: A* 알고리즘을 사용하여 캐릭터 AI 이동 구현중
    - 노수연: 상호작용 창 구현중(창 뜨는것 완료, 창 디자인만 입히면 안료)

    - 각자 맡은 부분에 대한 역할 상세 서술한 파일(한글파일) 및 현재까지 구현한 코드 파일(한글파일) 작성하여 4월 20일(목) 오후 8시까지 '최예린'에게 제출
    - 취합된 파일들은 정리 후 4명 각각에게 순서대로 파일을 도렬 수정 후 마지막 점검(최소 3개 이상씩 수정사항 발견하기)
    - 발표자료 수정 및 정리(피드백 반영) + 대본작성

3) ppt 추가 및 수정 역할 분배
    - 환경세팅 영상(김윤호): 현재 OS(WINDOWS 10 64bit) 창 보여주고, 유니티 설치(버전 확인), 프로젝트 생성하고 추가 설정 세팅, 비주얼 스튜디오 생성 후  C# 코드 작성 및 저장 후 유니티에 적용되고 실행에서 변경되는 것 까지 
    - 시나리오, 스토리(이하연): 인터넷에 게임 시나리오 제작과정을 참고하여 다시 제작
    - 화면 설계(최예린): 게임 구성과 상세화면, 전체적인 프로젝트 진핼 과정(시나리오 기획, 그래픽제작, 구현 , 테스트)
    - 알고리즘 정리(노수연): 다익스트라 알고리즘의 특징을 조사해서 비교 후 A* 알고리즘을 사용하는 이유를 자세히 적고 알고리즘 설명 부분 작성
